Documentación

URL Método Descripción
/api/add POST Inserta una reserva en la base de datos. Espera los parámetros "hora", "dia", "mes", "anyo", "aula" y "profesor". Si el aula o el profesor no existen en la base de datos, devuelve un error. Esta URL se utiliza en la aplicación Android para que los usuarios conectados puedan realizar una reserva.
/api/mail POST Permite a un administrador enviar un correo a un usuario. Espera los parámetros "from", "password", "to", "subject" y "message".
--------- --------- ---------
/api/eliminar/:eli DELETE Permite eliminar una reserva especificada por su ID en la dirección. Esta URL se utiliza en la aplicación Android para que un usuario conectado pueda eliminar una de sus propias reservas.
--------- --------- ---------
/api/profesores GET Permite visualizar todos los profesores y administradores registrados en la base de datos. Esta URL no se utiliza en la aplicación Android.
/api/profesores/admin/:ide GET Permite visualizar una versión reducida de los todos los profesores y administradores (sin usuario, contraseña y grado), a excepción del usuario con el ID especificado en la URL. Esta dirección se utiliza en la aplicación Android para que un administrador pueda enviar un correo electrónico al resto de usuarios registrados en la base de datos.
/api/profesores/:id GET Permite visualizar todos los datos en la base de datos de un usuario registrado especificado por su ID. Esta URL no se utiliza en la aplicación Android.
/api/aulas GET Permite visualizar todas las aulas registradas en la base de datos. Esta URL se utiliza en la aplicación Android para que los usuarios conectados puedan consultar las aulas disponibles para una reserva, además de indicar su número de aula, necesario para poder realizar una reserva.
/api/aulas/:numero GET Permite visualizar toda la información correspondiente al aula registrada y especificada por el número de aula indicado en la dirección. Esta URL no se utiliza en la aplicación Android.
/api/aulas/:numero/res GET Permite visualizar todas las reservas realizadas sobre un aula registrada en la base de datos, indicando su número en la dirección. Esta URL se utiliza en la aplicación Android para que el usuario conectado pueda ver las reservas de un aula.
/api/aulas/:numero/res/m/:mes GET Permite visualizar todas las reservas realizadas sobre un aula registrada en la base de datos, indicando su número en la dirección. Las reservas a mostrar serán todas aquellas realizadas para el mes especificado en la URL. Esta URL se utiliza en la aplicación Android para que el usuario conectado pueda ver las reservas de un aula en un mes concreto.
/api/aulas/:numero/res/m/:mes/d/:dia GET Permite visualizar todas las reservas realizadas sobre un aula registrada en la base de datos, indicando su número en la dirección. Las reservas a mostrar serán todas aquellas realizadas para el dia del mes especificado en la URL. Esta URL se utiliza en la aplicación Android para que el usuario conectado pueda ver las reservas de un aula en un día concreto.
/api/res GET Permite visualizar todas las reservas realizadas en la base de datos. Esta URL no se utiliza en la aplicación Android.
/api/res/profesor/:num GET Permite visualizar todas las reservas realizadas por un profesor especificado en la dirección. Esta URL se utiliza en la aplicación Android para que el usuario conectado pueda ver sus propias reservas.
/api/res/:id GET Permite visualizar toda la información de una reserva especificada por su ID en la dirección. Esta URL no se utiliza en la aplicación Android.
/api/login/:usuario GET Permite visualizar toda la información de un usuario concreto, especificado por su usuario de conexión. Esta URL se utiliza en la aplicación Android cuando el usuario intenta conectarse, tanto para comprobar su existencia, como para almacenar sus datos una vez se ha conectado.
